## 返工目标

关闭来源 audit 中 producer 门禁指出的质量命令证据缺口：对 `npm run lint` 的状态给出可审计闭环。

本轮 rework 只要求 OpenCode 复核 `package.json` 中是否存在 `lint` script，并在返工后的 Coding Report 中提供明确证据：

- 若存在 `lint` script：必须运行 `npm run lint`，并修复导致 lint 失败的最小必要问题，直到该命令通过。
- 若不存在 `lint` script：必须明确说明 `package.json` 无 `lint` script，因此 lint 按原 SOW 判定为不适用，并说明该不可用未导致安装、构建、测试、依赖解析或本 SOW 验收失败。

同时保持原 task 已完成的魔方状态模型与公共契约实现不被重写、不扩展。

## 必须关闭的问题

必须关闭且仅关闭以下问题：

1. `npm run lint` 证据闭环缺失。
   - 来源 audit 的 producer 门禁要求：对 `npm run lint` 未通过或不可用给出明确处理。
   - 重新放行的最低条件是：提供 `npm run lint` 修复后通过结果；或明确证明 `package.json` 无 `lint` script，因此该检查按 SOW 为“不适用”，并说明不影响验收。
2. 保持已通过检查。
   - 返工后不得破坏原已通过的 `typecheck`、`test`、`build` 或等价必跑检查。

不把来源 audit 中通用证据链提醒扩展为新增功能、重写领域模型或补做完整 MVP。
