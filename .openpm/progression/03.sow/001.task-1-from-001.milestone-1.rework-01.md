## 修改范围

OpenCode 按以下顺序执行；不得再次扩大为后续功能开发。

1. 仓库事实识别

   - 先识别当前真实项目结构：
     - 包管理器与 lockfile。
     - `package.json` 是否存在、是否 tracked、当前 scripts 和依赖。
     - 是否已有 `src/`、前端入口、构建配置、测试配置、`tsconfig.json`、lint 配置。
   - 若已有可用前端工程约定，必须沿用现有约定。
   - 若没有既有工程基线，允许新增最小 TypeScript 浏览器前端工程基线，但必须保持最小依赖、TypeScript strict、脚本可重复运行。
   - 若发现需要引入重型框架、真实 3D 引擎、复杂 E2E、长期不可逆技术选型或与 DESIGN/MSC 冲突，必须停止并报告。

2. 最小工程基线

   在没有既有工程基线时，允许新增或修改以下类别文件：

   - `package.json`
     - 增加可运行的最小 scripts，例如：
       - `build`
       - `typecheck`
       - `test`
       - `lint` 仅在本轮明确建立 lint 基线时新增。
     - 增加关闭本轮 audit 阻塞项所必需的最小依赖或 devDependencies。
     - 每个新增依赖必须在 Coding Report 中说明用途、替代方案、必要性和维护风险。
   - TypeScript 配置
     - 若新增 `tsconfig.json`，必须启用 `strict`。
     - 不得通过关闭 strict、跳过类型检查或放宽配置通过门禁。
   - 构建配置
     - 仅建立能构建最小浏览器页面的配置。
     - 不新增部署配置、后端配置或复杂资源管线。
   - 测试配置
     - 仅建立能运行当前 UI smoke 的最小测试配置。
     - 不引入复杂端到端测试体系，除非停止并获 Architect/用户确认。
   - 前端入口
     - 新增最小 `src/` 入口、UI shell 和样式文件。
     - `src/` 仅承载 UI shell、基础 DOM/组件组装、样式和测试辅助。

3. 最小 UI 骨架

     - 不得伪装为真实 3D 渲染已完成。
   - 页面显示“随机打乱”按钮。
   - 页面显示“求解”按钮。
   - 页面显示“速度”或等价含义的自动转动速度控件。
   - 页面显示“3 阶”“4 阶”“5 阶”切换入口。

## 风险

1. 范围膨胀风险  
   为关闭工程基线缺口，执行阶段可能误扩展到真实 3D、领域模型、打乱、求解、交互或动画。本 SOW 已将这些能力列为不做；如执行发现必须涉及，必须停止并交 Architect/Producer 复核。

2. 技术栈固化风险  
   构建工具、测试框架、`src/` 入口和 package scripts 一旦落地，会成为后续工程 contract。必须采用“已有优先、最小新增”，并触发 ADR 或 ADR 候选记录要求。

3. 依赖膨胀风险  
   新增依赖可能带来包体、兼容性和维护成本。不得为静态 UI 骨架引入重型 3D 引擎、复杂组件库或复杂 E2E 工具；每个新增依赖必须说明必要性。

4. TypeScript strict 风险  
   当前 audit 显示仓库未发现 `tsconfig.json`。若本轮建立 TypeScript 工程，必须保持 strict mode，不得通过关闭 strict 或跳过类型检查完成任务。

5. UI placeholder 误导风险  
   本轮只允许静态或低保真 UI 骨架。必须通过文案、disabled 状态或提示明确真实 3D、随机打乱、求解和自动播放尚未实现，避免用户误解。

6. 领域边界风险  
   默认 3 阶、计数器 `0`、速度控件等只能作为 UI 展示状态，不得固化为 `CubeState`、`Move`、`RunMode` 等领域公共契约。后续领域状态仍必须遵守 MSC：`CubeState` 由魔方逻辑状态模型唯一拥有，渲染层只派生视觉状态。

7. 工作区归档风险  
   上一轮 audit 已发现未跟踪 `package.json` 与 `.openpm/progression` 目录。下一轮必须用 git status、关键 diff 和文件清单说明变更范围，避免产品代码、配置文件与项目管理 artifact 混淆。

8. ADR 遗漏风险  
   若本轮落地工程基线但未记录 ADR 或 ADR 候选，会导致长期工程事实来源不清。若无法在本轮修改 ADR，Coding Report 必须把 ADR 补齐列为进入 commit 前的待办/阻塞项。
