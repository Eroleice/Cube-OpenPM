## 返工目标

- Rework 轮次：01。
- 关闭当前 audit 判定为“需要返工”的最小阻塞项：`tests/app.smoke.test.ts` 未证明 DOM 环境可用，也未证明入口挂载链路可执行。
- 返工完成后，后续 code -> audit 应能基于真实源码、测试、命令和 git 证据确认：工程基线 smoke 测试覆盖测试框架、DOM 环境和入口模块可执行性。
- 本轮不改变原项目业务目标，不进入 3D 魔方业务 UI、状态模型、渲染、交互、打乱或求解能力实现。

## 必须关闭的问题

1. **smoke 测试覆盖不足**
   - Audit finding：`tests/app.smoke.test.ts` 当前仅调用 `App()` 并断言字符串，不能证明 DOM 环境可用，也不能证明入口挂载链路可执行。
   - 必须修复为：测试在 Vitest DOM 环境中创建 DOM 容器，执行入口挂载链路，例如调用 `renderApp(container)` 或等价挂载函数，并断言容器中出现用户可见基线文案或稳定根节点，例如 `data-testid="app-root"`。

2. **入口挂载链路证据不足**
   - 必须让测试能够证明应用入口的最小挂载能力可执行。
   - 优先采用可控的挂载函数测试方式；如选择直接导入或验证 `src/main.ts`，必须先准备 `#app` 容器，并避免模块副作用污染其他测试。

3. **审计证据链不足**
   - 下一轮 Coding Report 必须逐条补齐源码、测试、命令、git status、diff、未跟踪文件和 OpenPM artifact 证据。
   - 不得只用 OpenCode success、退出码 0 或“测试通过”的笼统摘要替代可审计证据。

## 修改范围

允许修改的最小范围：

- `tests/app.smoke.test.ts`
  - 修复 smoke 测试，使其创建 DOM 容器并验证挂载结果。
  - 测试断言必须覆盖以下至少一项：
    - 用户可见基线文案出现在 DOM 中；
    - 稳定根节点存在，例如 `data-testid="app-root"`。

- `src/` 内与入口挂载相关的最小衔接文件
  - 仅当现有入口没有可测试挂载函数时，允许最小调整或导出 `renderApp(container)` / 等价函数。
  - 调整必须保持入口职责清晰：渲染基线应用内容到指定 DOM 容器，不引入业务 UI 或 3D 魔方逻辑。

- 测试环境或工程配置文件
  - 仅当现有 Vitest DOM 环境无法支持该 smoke 测试时，允许最小修改 `vitest.config.ts`、`vite.config.ts`、`package.json` 或相关配置。
  - 如修改配置，Coding Report 必须说明必要性、影响范围和未改变 TypeScript strict 约束。

- `.openpm/progression/00.documents/ADR.md`
  - Audit 提到 ADR 证据需闭环。本轮若已有工程基线 ADR 记录，则 Coding Report 必须引用其 ADR 编号；若缺失且本轮确实补齐工程基线决策记录，则允许最小更新 ADR。
  - ADR 编号要求：下一轮 Coding Report 必须列出被引用或新增的 ADR 编号；若新增，建议使用仓库既有 ADR 编号格式，若无既有编号格式则使用 `ADR-0001` 记录工程基线、包管理器、测试环境和入口挂载策略。

## 不做

- 不新增 3D 引擎、求解库、运行时业务依赖或大版本依赖升级。
- 不重构项目架构、公共 API、资源管线、部署策略或目录结构。
- 不修改 milestone、业务 task 或 DESIGN/MSC 的产品范围。
- 不通过关闭 TypeScript strict、放宽 `tsconfig`、跳过类型检查或降低测试断言强度规避失败。
- 不因 packageManager 声明版本与本地版本可能存在差异而主动对齐版本：当前 audit 未指出该差异导致安装、构建、测试、依赖解析或 SOW 验收失败；若下一轮真实命令证明该差异造成失败，才允许在最小范围内对齐并说明原因，否则仅记录为风险或后续对齐项。

## 风险

- **范围滑坡风险**：本轮 audit finding 只指向 smoke 测试、DOM 环境、入口挂载链路和证据链，执行方不得扩展到业务 UI 或 3D 魔方能力。
- **测试脆弱风险**：直接导入 `src/main.ts` 可能带来模块副作用和 DOM 容器准备顺序问题；优先使用可控的 `renderApp(container)` 或等价挂载函数测试。
- **证据链风险**：如果下一轮 Coding Report 仍只提供摘要，缺少测试断言、命令结果、git status/diff 或未跟踪文件说明，QA 仍可能判定需要返工或阻塞。
- **配置边界风险**：若为通过测试而修改 Vitest/Vite/package 配置，必须保持最小化；若触及构建边界、公共 API 或测试环境策略变化，应升级 Architect 复核。
- **packageManager 风险**：当前未证明 packageManager 声明版本与本地版本差异造成失败，不要求主动对齐；若下一轮命令显示该差异导致安装、构建、测试、依赖解析或验收失败，再作为最小修复处理。
- **提交风险**：在后续 QA audit 结论为通过或有条件通过前，不得进入 commit。
