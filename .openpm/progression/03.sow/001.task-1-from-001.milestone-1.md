- 来源 Task：.openpm/progression/02.task/001.task-1-from-001.milestone-1.md

## 施工目标

建立浏览器页面入口与基础 UI 初始状态，让用户打开页面后能看到 3D 魔方程序的最小界面骨架。

本次施工只交付用户可见的初始 UI 骨架与 smoke 覆盖，具体包括：

- 可识别的 3D 魔方展示/承载区域，可以是明确占位容器，不要求真实 3D 几何渲染。
- “随机打乱”按钮。
- “求解”按钮。
- “速度”或等价含义的自动转动速度滚动条。
- “3 阶”“4 阶”“5 阶”切换入口。
- 打乱步骤数计数器，初始显示 `0` 或等价明确空状态。
- 求解步骤数计数器，初始显示 `0` 或等价明确空状态。
- 默认 UI 状态明确表达为 3 阶选中或等价默认 3 阶初始状态。
- 未完成能力必须有 disabled、占位提示、title/aria 描述或可见 no-op 反馈，不能表现为“可点击但无反馈”的已完成能力。

本 task 追溯 DESIGN 验收口径 A3-A9，并对 A1/A2 仅交付“魔方承载区域/完成态占位表达”，不交付真实 3D 完成态魔方。

## 当前事实

当前仓库已完成最小 Vite/TypeScript/Vitest 工程基线，已读取到以下事实：

- `package.json` 存在，项目名为 `cube-openpm`，`type` 为 `module`。
- `package.json` 声明 `packageManager` 为 `npm@10.8.2`。
- 现有 scripts：
  - `npm run dev` → `vite`
  - `npm run build` → `vite build`
  - `npm run typecheck` → `tsc --noEmit`
  - `npm run test` → `vitest run`
- 现有 devDependencies 包含 `typescript`、`vite`、`vitest`、`jsdom`、`@types/node`。
- `tsconfig.json` 存在，`strict: true`，`include` 覆盖 `src`、`tests`、`vite.config.ts`、`vitest.config.ts`。
- 浏览器入口链路：
  - `index.html` 预计通过 `/src/main.ts` 挂载。
  - `src/main.ts` 导出 `renderApp(container: HTMLElement)` 并自动调用 `mountFromDocumentRoot()`。
  - `src/App.ts` 当前只输出 baseline 文案：`Cube OpenPM baseline ready.`。
- 现有 smoke 测试：`tests/app.smoke.test.ts`，当前只验证 baseline 根节点和 baseline 文案，不覆盖本 task 的用户可见 UI 骨架。
- 当前没有真实 3D 渲染、`CubeState`、`Move`、`MoveSequence`、`RunMode`、`PlaybackState`、随机打乱、求解、自动播放或阶数切换业务闭环。

架构边界事实：

- DESIGN 要求页面初始展示魔方主体、基础控件、计数器和速度滚动条。
- MSC 要求 UI 控制与状态展示模块不直接修改 `CubeState`，不自行推断步骤数，不直接驱动 3D 动画，不实现求解算法。
- 本 task 允许 UI 临时展示“默认 3 阶”和两个计数器初始值，但不得把这些临时 UI 表达升级为魔方逻辑状态权威。
- 本 SOW 默认不需要新增 ADR，因为它只修改 UI shell、样式和 smoke 测试，不新增公共契约、schema、API、持久化格式、部署假设或第三方 3D/求解依赖；如 OpenCode 实际需要触及这些内容，必须停止扩大范围并按 ADR owner：Architect 处理。

   - 速度控件本 task 只展示默认值和入口，不要求驱动自动播放速度；应有占位说明或 disabled 语义，避免误导。

5. 补齐最小样式
   - 可以新增或调整最小 CSS，使魔方区域、控制区、计数器和阶数入口在页面上可识别。
   - 样式只服务初始 UI 可读性，不做完整视觉设计、主题皮肤、动画或响应式优化。

6. 更新 smoke 测试
   - 更新或新增 smoke 测试，验证页面初始渲染后的用户可见结果。
   - smoke 断言应围绕文本、role、label、accessible name 或稳定 `data-testid`，避免依赖脆弱 DOM 层级。
   - 测试必须覆盖本 SOW “测试要求”和“验收标准”列出的初始 UI 项。

7. 配置与 ADR 限制
   - 默认不得修改 `package.json`、lockfile、`tsconfig.json`、Vite/Vitest 配置或 ADR。

6. 未完成功能不误导用户

## 风险

主要风险与控制方式如下：

- 范围滑坡风险：OpenCode 可能顺手实现真实 3D、打乱、求解、自动播放、阶数切换闭环或公共领域契约。控制方式：严格按“不做”执行；若必须触及公共契约或 3D 库选型，停止并升级 Architect。
- UX 误导风险：按钮可点击但无反馈会让用户误以为功能已完成。控制方式：优先 disabled 或明确占位提示；如 no-op 必须有可见且可测试反馈。
- 测试证据不足风险：仅构建通过无法证明页面初始 UI 达标。控制方式：必须更新 smoke 测试，直接断言用户可见 DOM、accessible name 或明确文本。
- 架构边界污染风险：UI 临时状态可能演变为 `CubeState` 替代权威。控制方式：本 task 只允许展示默认 3 阶和初始计数器，不引入魔方逻辑状态模型。
- 工程基线误改风险：为 UI smoke 修改 package、tsconfig 或测试配置可能影响后续任务。控制方式：默认不改配置；若必须改，需说明原因、影响，并按 ADR owner：Architect 治理。
- 包管理器版本差异风险：`package.json` 声明 `npm@10.8.2`，本地 npm 版本可能不同。控制方式：Coding Report 必须报告差异和影响；不影响验收时不作为阻塞，影响安装/构建/测试时要求后续 rework 对齐。
- 可访问性与可测性风险：控件缺少 label 或文本会导致 smoke 依赖脆弱选择器。控制方式：关键控件必须有文本、label、aria 标识或 accessible name。
- 后续任务衔接风险：当前魔方区域只是占位，后续 task 仍需基于 `CubeSize` 与 `CubeState` 完成真实 3D 完成态渲染。控制方式：文案和代码注释避免宣称真实渲染已完成。
