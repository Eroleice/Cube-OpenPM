# 002.task-2-from-001.milestone-1

- 来源 Milestone: .openpm/progression/01.milestone/001.milestone-1.md

## 目标

建立魔方状态模型与公共契约的最小可测试边界，为后续完成态渲染、阶数切换、手动转动、随机打乱和自动求解提供统一基础。

本 task 需要定义并使用 `CubeSize`、`CubeState`、`Move` / `MoveSequence`、`RunMode` / `PlaybackState` 的最小公共语义，其中 `CubeState` 是魔方逻辑状态唯一权威来源。

## 独立验收标准

- 存在 `CubeSize` 公共类型或等价公共契约，并且仅允许 `3 | 4 | 5`。
- 非法阶数不能只依赖 UI 选项规避，必须在状态模型或公共入口处有可测试的收敛方式。
- 存在 `CubeState` 完成态创建能力，能基于合法 `CubeSize` 创建对应阶数的完成态魔方逻辑状态。
- 存在 `CubeState` 完成态判断能力，能判断新建完成态为已完成。
- `CubeState` 被明确定位为魔方逻辑状态唯一权威来源；UI、输入模块、渲染模块不得直接改写魔方逻辑状态。
- 存在 `Move` / `MoveSequence` 的最小公共语义边界，为后续手动转动、随机打乱和求解共用；但不实现完整转动算法、随机打乱算法或求解算法。
- 存在 `RunMode` / `PlaybackState` 的最小状态边界，至少覆盖 `idle`，并为后续自动流程保留扩展空间。
- 状态模型、阶数合法性、完成态创建和完成态判断具备单元测试或等价可测试证据。
- 公共契约 owner 清晰，不能归属于渲染层或 UI 层。

## 建议切片大小

适合一次 SOW/code/audit 循环完成。

建议范围控制在纯领域状态与公共类型边界，不跨入真实 3D 渲染、不跨入 UI 编排、不实现后续算法。若发现现有源码已有品牌化 ID、Result/AppError、schema 或共享类型模式，后续 SOW 应优先复用项目已有模式。

## 不做

- 不实现页面 UI 控件或布局。
- 不实现真实 3D 完成态渲染。
- 不实现阶数切换 UI 编排。
- 不实现鼠标右键视角旋转。
- 不实现鼠标左键拖动层转动。
- 不实现随机打乱 20 步自动播放。
- 不实现自动求解。
- 不实现完整转动算法或动画协议。
- 不承诺最优解。
- 不引入账号、云存储、排行榜、教学、移动端触摸、主题皮肤。
- 不支持 3/4/5 阶以外魔方。
- 不做完整性能优化。
- 不在本 task 中做出难迁移的 `CubeState` 内部表达或 `Move` 语义选择；如后续实现必须锁定难迁移方案，应交由 Architect 复核并补充 ADR 候选。

## 后续 SOW 备注

- 后续 SOW 必须确认源码现有模块边界，优先把共享公共契约放在项目合适的领域状态或 shared contract 位置，避免放入 UI 或渲染模块。
- 如果项目暂无清晰结构，共享纯类型、序列化辅助、品牌化 ID、Result/AppError 等跨模块基础代码可建议进入 `src/shared`，但必须先确认项目没有更合适约定。
- 单元测试应覆盖合法阶数、非法阶数收敛、完成态创建、完成态判断、`idle` 初始运行状态等关键边界。
- 若 `CubeState` 表达、`Move` 语义或播放状态机选择影响后续兼容性，应在 SOW 中标注 Architect 复核和 ADR 判断。
- 本 task 追溯 milestone 验收项：`CubeSize` 合法性、`CubeState` 完成态创建/判断、`Move` / `MoveSequence` 最小语义、`RunMode` / `PlaybackState` 最小状态、状态模型测试证据、`CubeState` 权威边界。
