# 004.task-4-from-001.milestone-1

- 来源 Milestone: .openpm/progression/01.milestone/001.milestone-1.md

## 目标

实现 3 阶、4 阶、5 阶切换的最小闭环：用户选择阶数后，系统重建对应阶数的完成态 `CubeState`，更新 3D 完成态展示，并将打乱步骤数与求解步骤数重置为 `0` 或等价明确空状态。

本 task 聚焦 UI、领域状态与渲染入口之间的编排，不重新定义状态模型或渲染架构。

## 独立验收标准

- 用户可以通过页面上的“3 阶”“4 阶”“5 阶”入口选择当前阶数。
- 选择 3 阶后，魔方处于 3 阶完成态并展示 3 阶完成态主体。
- 选择 4 阶后，魔方处于 4 阶完成态并展示 4 阶完成态主体。
- 选择 5 阶后，魔方处于 5 阶完成态并展示 5 阶完成态主体。
- 每次切换阶数后，必须通过领域状态能力重建对应阶数完成态 `CubeState`，不得复用上一阶内部对象后局部修补。
- 每次切换阶数后，打乱步骤数重置为 `0` 或等价明确空状态。
- 每次切换阶数后，求解步骤数重置为 `0` 或等价明确空状态。
- 切换后不保留上一阶魔方内部状态。
- 切换后不继续展示或播放上一阶魔方视觉状态；旧视觉对象应被清理或替换。
- 计数器状态不得归渲染层所有。
- 具备 smoke、组件测试或单元测试证据，覆盖阶数切换、状态重建、计数器归零和旧状态不残留的关键断言。

## 建议切片大小

适合一次 SOW/code/audit 循环完成。

建议范围控制在阶数切换编排、状态重建、计数器重置、渲染刷新或替换触发以及对应测试证据。该 task 应依赖 task 1 的 UI 入口、task 2 的状态模型、task 3 的渲染入口，不应在本 task 中重新实现这些基础能力。

## 不做

- 不重新定义 `CubeSize`、`CubeState`、`Move` / `MoveSequence`、`RunMode` / `PlaybackState`。
- 不重写 3D 完成态渲染架构。
- 不实现鼠标右键视角旋转。
- 不实现鼠标左键拖动层转动。
- 不实现随机打乱 20 步自动播放。
- 不实现自动求解。
- 不实现完整动画协议。
- 不承诺最优解。
- 不引入账号、云存储、排行榜、教学、移动端触摸、主题皮肤。
- 不支持 3/4/5 阶以外魔方。
- 不做完整性能优化。
- 不改变 milestone 默认规则：切换阶数后计数器重置为 `0`。

## 后续 SOW 备注

- 后续 SOW 必须明确本 task 是编排 task：复用既有 UI 控件、状态模型和渲染入口，不允许在 UI 或渲染层绕过 `CubeState` 直接维护魔方逻辑。
- 测试应覆盖至少一次 3→4、4→5、5→3 或等价切换路径，并断言计数器重置和旧状态不残留。
- 若当前实现中存在临时 UI 状态，应在本 task 中收敛为对领域状态创建能力的调用或等价公共入口，不应扩大临时状态权威。
- 如发现切换规则需要从“重置为 0”变更为保留历史计数，必须交由 Producer 复核，不得在实现中自行改变产品规则。
- 本 task 追溯 milestone 验收项：3/4/5 阶切换入口、切换后对应完成态、计数器归零、不保留上一阶状态、不继续展示或播放上一阶视觉状态。
