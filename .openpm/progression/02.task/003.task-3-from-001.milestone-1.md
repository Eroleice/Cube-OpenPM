# 003.task-3-from-001.milestone-1

- 来源 Milestone: .openpm/progression/01.milestone/001.milestone-1.md

## 目标

基于 `CubeSize` 与 `CubeState` 渲染 3 阶、4 阶、5 阶完成态 3D 魔方主体，并固化渲染层与逻辑状态的边界：渲染层只消费状态，不成为业务状态源。

本 task 的交付价值是让用户能够看到与当前阶数对应的完成态 3D 魔方结构和各面颜色状态，为 milestone 的最小可见闭环提供核心视觉主体。

## 独立验收标准

- 页面魔方展示区域能基于 `CubeSize` 与 `CubeState` 显示完成态 3D 魔方主体。
- 对 3 阶、4 阶、5 阶输入，渲染结果能表达对应阶数结构；至少应能通过 smoke、组件测试、DOM/可访问文本、渲染容器结构或等价可审计证据区分当前阶数。
- 完成态魔方能表达各面颜色状态，且颜色状态来源于 `CubeState` 或其可追溯派生数据。
- 渲染层只消费 `CubeSize` 与 `CubeState`，不得维护可被业务读取或作为权威使用的魔方逻辑状态。
- UI、输入模块、渲染模块不得直接改写 `CubeState`。
- 渲染对象生命周期需为后续阶数切换预留清理或替换能力，避免旧视觉对象长期残留。
- 打开页面后用户能看到 3D 魔方主体；在与 task 1、task 2 集成后，默认应显示 3 阶完成态主体。
- 如实施需要新增第三方 3D 渲染库，必须说明依赖用途、替代方案、包体风险、浏览器兼容性风险、维护风险，并交由 Architect 复核；不得在 task 阶段默认批准具体库。

## 建议切片大小

适合一次 SOW/code/audit 循环完成。

建议范围控制在完成态静态 3D 展示和渲染边界，不实现用户交互、转层动画、随机打乱播放、求解播放或完整动画协议。若仓库已有渲染库或前端框架，应优先沿用现有技术栈；若必须新增依赖，需在后续 SOW 中触发 Architect 复核。

## 不做

- 不重新定义 `CubeSize`、`CubeState`、`Move` / `MoveSequence`、`RunMode` / `PlaybackState` 公共契约；应复用 task 2 产物。
- 不实现阶数切换业务闭环和计数器重置编排。
- 不实现鼠标右键视角旋转。
- 不实现鼠标左键拖动层转动。
- 不实现随机打乱 20 步自动播放。
- 不实现自动求解。
- 不实现完整动画协议。
- 不承诺最优解。
- 不引入账号、云存储、排行榜、教学、移动端触摸、主题皮肤。
- 不支持 3/4/5 阶以外魔方。
- 不做完整性能优化，只要求 3/4/5 阶完成态基本可展示。
- 不把渲染对象作为业务状态源。

## 后续 SOW 备注

- 后续 SOW 必须先核实现有前端框架、构建入口和是否已有 3D 渲染依赖。
- 如果新增第三方库，应明确 Architect 复核点，并判断是否需要 ADR；如该库影响长期渲染协议、资源组织或部署假设，应补充 ADR 候选。
- 验收证据可组合使用 smoke、组件级测试、渲染容器存在性检查、状态输入到渲染输出的可审计断言；不应完全依赖人工截图。
- 渲染实现应保持与领域状态单向依赖：领域状态不依赖渲染，渲染消费领域状态。
- 本 task 追溯 milestone 验收项：打开页面可见 3D 魔方主体、3/4/5 阶完成态渲染、各面颜色状态、渲染层不成为业务状态源、第三方渲染依赖风险说明。
