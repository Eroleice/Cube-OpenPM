# 001.task-1-from-001.milestone-1

- 来源 Milestone: .openpm/progression/01.milestone/001.milestone-1.md

## 目标

建立浏览器页面入口与基础 UI 初始状态，让用户打开页面后能看到 3D 魔方程序的最小界面骨架：魔方展示区域、基础操作控件、阶数切换入口、速度控件和计数器初始状态。

本 task 聚焦用户可见的页面初始体验，不要求真实 3D 魔方渲染完成，也不要求按钮具备打乱或求解能力。允许魔方区域在本 task 中使用明确的占位容器，但不得固化与后续 `CubeSize`、`CubeState` 冲突的业务状态结构。

## 独立验收标准

- 打开页面后能看到一个用于承载 3D 魔方主体的可识别区域。
- 页面显示以下基础控件或入口：
  - “随机打乱”按钮。
  - “求解”按钮。
  - “速度”或等价含义的自动转动速度滚动条。
  - “3 阶”“4 阶”“5 阶”切换入口。
  - 打乱步骤数计数器。
  - 求解步骤数计数器。
- 默认 UI 状态表达为 3 阶选中或等价的默认 3 阶初始状态。
- 初始打乱步骤数和求解步骤数显示为 `0`，或有等价明确空状态。
- “随机打乱”“求解”“速度”在本 milestone 中仅作为基础控件展示，不得误导用户认为随机打乱、自动求解或自动播放已完成；允许 disabled、占位提示或 no-op，具体行为留给 SOW 阶段确定。
- 用户可见初始页面具备 smoke 覆盖要求，至少能证明基础控件、计数器和魔方承载区域可见。
- 本 task 不引入 UI、输入模块或渲染模块直接改写魔方逻辑状态的设计；如需要临时 UI 状态，只能用于页面展示，不得成为 `CubeState` 的替代权威。

## 建议切片大小

适合一次 SOW/code/audit 循环完成。

建议范围控制在页面入口、基础布局、初始控件展示、初始计数器文本和最小 smoke 覆盖内。若仓库尚无前端入口，后续 SOW 需先确认真实技术栈、包管理器、构建入口和测试框架，再按项目现有结构建立最小入口。

## 不做

- 不实现真实 3D 完成态魔方渲染。
- 不实现 `CubeState`、`Move`、`MoveSequence`、`RunMode` 或 `PlaybackState` 的公共契约。
- 不实现 3/4/5 阶切换后的状态重建闭环。
- 不实现鼠标右键视角旋转。
- 不实现鼠标左键拖动层转动。
- 不实现随机打乱 20 步自动播放。
- 不实现自动求解。
- 不承诺最优解。
- 不引入账号、云存储、排行榜、教学、移动端触摸、主题皮肤。
- 不支持 3/4/5 阶以外魔方。
- 不做完整性能优化。
- 不在 task 阶段预批准任何具体第三方 3D 渲染库。

## 后续 SOW 备注

- 后续 SOW 必须先识别仓库真实技术栈、目录结构、入口文件、包管理器、测试框架和已有脚本。
- 如项目为 TypeScript，必须保持 strict mode，不得通过关闭 strict、跳过类型检查或放宽配置完成任务。
- smoke 覆盖应聚焦“页面打开后基础 UI 可见”，不要求本 task 验证真实 3D 几何结构。
- UI 骨架可先行，但后续 task 必须以领域状态模块的 `CubeSize` 与 `CubeState` 作为业务状态基础，不允许 UI 临时状态演变成魔方逻辑状态权威。
- 本 task 追溯 milestone 验收项：页面初始加载、基础控件可见、计数器初始状态、默认 3 阶 UI 表达、用户可见初始页面 smoke 覆盖。
