# Agent Role Card 通用规则

## 通用执行边界

- Agent 阶段只做分析、设计、交接、审查或调度；不得声称已经真实修改代码、运行测试或完成落盘，除非该动作由本地工具真实完成并有记录。
- 如果需要真实改文件、运行命令或写执行报告，应交给 OpenCode 或对应本地执行环境。
- 输出必须区分当前仓库事实、文档蓝图、Agent 推断和建议执行项。
- 交接给执行器时必须写清目标文件、允许范围、禁止范围、验收标准、必跑检查和风险。
- 如果只是分析、设计或审查，必须明确说明无需真实文件修改。

## 通用升级规则

- 跨模块、schema、API、数据流、权限、部署或 public contract 变化交给 Architect 复核。
- 里程碑、优先级、范围拆分、延期项或交付风险交给 Producer 复核。
- 用户可见流程、交互、可访问性或错误反馈风险交给 Designer 或 QA 复核。
- 回归风险、测试缺口、审计结论或质量门禁交给 QA 复核。
- 文档一致性、ADR、项目记忆或 artifact 契约问题交给当前阶段负责角色复核，架构类问题交给 Architect 复核。

## 通用输出要求

- 不要输出代码围栏，除非当前 artifact 明确需要展示代码片段。
- 不要把未验证事实写成“已实现”。
- 项目记忆更新只写两类内容：`本次完成` 用一句话概括本次 workflow 并附相关 artifact 路径；`后续对齐` 写后续功能或模块开发必须继续遵守的长期约束、接口边界、风险或决策。没有后续影响时写 `无`。
